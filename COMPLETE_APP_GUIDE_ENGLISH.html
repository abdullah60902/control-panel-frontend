üíä Part 4: Medication Module (eMAR)

What is eMAR?

Electronic Medication Administration Record - A digital MAR chart that replaces paper-based medication records.

Conducting a Medication Round

  1. Open Medication section from sidebar
  2. Select time slot: Morning (8am), Afternoon (2pm), Evening (6pm), Night (10pm)
  3. Medicine list appears for each resident
  4. After administering medicine, click GIVE
  5. If resident refuses, click REFUSE and enter reason

Medication Screen Buttons

Button Color What Happens
GIVE Green Medicine administered. Stock automatically reduced. Timestamp recorded.
REFUSE Red Resident refused medicine. Reason mandatory (e.g., "Sleeping", "Nausea")
Add Medicine Blue Add new prescription. Enter name, dosage, frequency, start date.
Edit Yellow Modify existing medicine details (dosage, timing)
Discontinue Red Stop medicine (requires doctor's order)
Stock Alert Orange Shows when stock is low. Reminder to reorder.
Print MAR Gray Download monthly MAR chart as PDF
‚ö†Ô∏è Important: For PRN (as needed) medications, entering a reason is mandatory before administration.

üìã Part 7: Incident Reports

When to Create an Incident Report

  • Resident fall
  • Injury (cut, bruise, etc.)
  • Medication error (wrong dose, missed dose)
  • Behavioral incident (aggression, wandering)
  • Equipment failure (hoist broke, bed malfunction)
  • Safeguarding concern (suspected abuse)

Creating an Incident Report

  1. Open Incident Reports from dashboard or sidebar
  2. Click + New Incident
  3. Fill in form: Incident Type, Date & Time, Location, Description, Witnesses, Action Taken
  4. Click Submit
  5. Manager receives automatic notification

Incident Report Buttons

Button Function
New Incident Create new incident report
View Details View full details of existing incident
Add Follow-up Update with actions taken after incident
Upload Photo Upload photo of injury or damage (evidence)
Mark Reviewed Manager has reviewed (approval)
Export PDF Print incident report (for insurance/legal)
6-Month Overview Chart showing incident trends over time
‚úì Best Practice: Complete incident reports within 24 hours. Inform family if it's a serious incident.

üéØ Summary: How the System Works

Daily Workflow Example

  1. Morning (8am): Login ‚Üí Check dashboard for alerts ‚Üí Medication round (click GIVE for each medicine) ‚Üí Write daily logs
  2. Afternoon (2pm): Lunch medication round ‚Üí Create incident report if needed ‚Üí Review care plans
  3. Evening (6pm): Evening medication round ‚Üí Write handover notes for next shift ‚Üí Complete compliance checks
  4. Night (10pm): Night medication round ‚Üí Final checks ‚Üí Logout

Key Points to Remember

  • Green Buttons = Positive actions (Save, Give, Complete)
  • Red Buttons = Negative/Delete actions (Refuse, Delete, Discontinue)
  • Blue Buttons = Add/Create new records
  • Yellow Buttons = Edit/Update existing records
  • Red Alerts = Urgent action required
  • Yellow Alerts = Warning, address promptly
